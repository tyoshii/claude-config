# /recap

「いまなんだっけ？」を素早く把握するためのコマンド。

一夜明けて作業を再開するとき、別の作業から戻ってきたときなどに使う。

## 実行手順

以下のすべてを並列で収集し、最後にまとめて報告する。

### 1. Git の状態

以下を実行して把握する：

- `git branch --show-current` — 現在のブランチ
- `git status --short` — 変更ファイルの一覧
- `git log --oneline -5` — 直近のコミット
- `git stash list` — スタッシュの有無

### 2. Worktree の確認

- `git rev-parse --git-dir` の結果に `.git/worktrees/` が含まれるなら worktree 内にいる
- worktree 内なら、メインの作業ディレクトリのパスも表示する
- `git worktree list` で他の worktree があるか確認する

### 3. PR の状況

- `gh pr status` で自分に関連する PR を確認する
- 現在のブランチに紐づく PR があるかも確認する

### 4. 直近のセッション履歴

Claude のセッションログから直近のユーザープロンプトを抽出する：

```bash
# 現在のプロジェクトのセッションログを特定
PROJECT_DIR=$(pwd | sed 's|/|-|g; s/^-//')
SESSION_DIR="$HOME/.claude/projects/$PROJECT_DIR"

# 最新のセッションログから直近のユーザープロンプトを抽出（最大10件）
if [ -d "$SESSION_DIR" ]; then
  for f in $(ls -t "$SESSION_DIR"/*.jsonl 2>/dev/null | head -5); do
    grep '"type":"user"' "$f" | tail -5 | jq -r '.message.content' 2>/dev/null | head -c 200
  done
fi
```

- 複数セッションのログがある場合は、更新日時が新しいものから優先
- ユーザーの発言のみ抽出する（`"type":"user"` のエントリ）
- 各プロンプトは先頭 200 文字程度に省略してよい

### 5. 夜間作業報告書の確認

- プロジェクトルートに `overnight-report*.md` やそれに類するファイルがないか確認する
- あれば内容を読んで報告に含める

## 報告フォーマット

収集した情報を以下の形式で簡潔に報告する：

```
## 現在地

- ブランチ: <branch>
- ディレクトリ: <cwd>
- Worktree: いいえ / はい（メイン: <path>）

## 変更状況

<git status の要約。なければ「クリーン」>

## 直近のコミット

<git log --oneline -5 の結果>

## PR

<関連する PR の一覧。なければ「なし」>

## 直近の作業内容

<セッションログから抽出した直近のプロンプトを要約>
「〇〇をお願いしていた」「△△の実装を進めていた」のように、
何をしようとしていたかを自然な言葉でまとめる。
```

## 注意

- このコマンドは情報収集のみ。ファイルの変更やコミットは一切行わない
- 報告は簡潔にまとめる。詳細が必要なら深掘りはユーザーが判断する
- セッションログが見つからない場合はその旨を伝え、Git 情報だけ報告する
