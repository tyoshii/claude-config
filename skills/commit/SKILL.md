# /commit [push]

変更内容を確認し、適切なコミットメッセージを作成してコミットを実行する。

## 引数

- `push` : コミット後に `git push` も実行する

## 実行手順

### 1. 現状確認

以下を実行して状況を把握する：

- `git status` で変更ファイルを確認
- `git diff` でステージ済み・未ステージの差分を確認
- `git log --oneline -10` で直近のコミットスタイルを確認

### 2. 言語の判定

コミットメッセージの言語を決定する：

- `config.yml` の `commit.language` を参照
- `auto` の場合：直近 5 件のコミットメッセージから判定
- 判定できない場合は日本語をデフォルトとする

### 3. コミットメッセージの作成

#### 基本ルール

- 直近のコミットスタイル（prefix、フォーマット）に寄せる
- 1 行目は変更の要約（50 文字以内を目安）
- 命令形または体言止めで統一

#### 変更規模による分岐

**小さな変更（1-2 ファイル、単一目的）：**
```
typo を修正
```
1 行のみで十分。本文は不要。

**複数の変更がある場合：**
```
ユーザー認証機能を追加

- ログイン API を実装
- セッション管理を追加
- 認証ミドルウェアを作成
```
本文で箇条書きにして変更点を列挙。

### 4. footer の付与

`config.yml` の `commit.footer` が `true` の場合のみ、以下を末尾に追加：

```
Co-Authored-By: Claude <noreply@anthropic.com>
```

`false` または未設定の場合は付与しない。

### 5. ステージング

関連する全ファイルを `git add` する。

### 6. コミット実行

- 確認は求めず、そのまま `git commit` を実行する

### 7. push（引数に `push` がある場合のみ）

- `git push` を実行する
- 引数がない場合は push しない

## 注意事項

- 特定のリポジトリ構造（docs/、src/ など）に依存しない
- conventional commits を強制しない（リポジトリの慣習に従う）
- 機密情報を含むファイルがステージされていないか確認する
- `.env`、`*.secret`、`credentials` などは警告を出す
